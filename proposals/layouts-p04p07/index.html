P-04 / P-07 レイアウト提案

PC で 1画面内に収まる構成を 3 パターン用意しました。
各パターンとも P-04（企業・選考）と P-07（ポートフォリオ）の両方に同じ構造を適用しています。

共通の制約

  • PC モーダル想定: 90vw × 90vh（最小 1280 × 720 程度）でスクロールなし完結
  • 既存 v1.1 のデザイントークン（漆ガラス・カラーパレット・フォント）を踏襲
  • P-07 は履歴書機能（基本情報・自己PR・学歴・職歴・資格・スキル・履歴・バッジ）を全て収める
  • P-04 はカリキュラム・ロードマップ・選考・受講中コース・ゲートを全て収める

選定の観点

Pattern B はモーダル内/モバイルで安定、ただし「全体を一目で見せる」用途には弱い。
Pattern A と C は 1 画面で全情報を見せる構成だが、A は均等密度・C は強弱あり。
P-07（ポートフォリオ＝履歴書）は 面接官に印象を残す 観点で C が強く、P-04（企業・選考＝進捗確認）は 状況把握 観点で A が強い、という分け方も検討余地あり。

フィードバックを元にどのパターン（または組み合わせ）で進めるか決定 → 本番モックに反映の流れになります。
