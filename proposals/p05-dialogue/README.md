## 公開URL

`https://thenewgate-inc.github.io/cytech-urushi-os-mockup/proposals/p05-dialogue/`

## 提案の背景

v1.1 の P-05 は教科書型レイアウト（左に長文、右にAI＋目次）。社内から **「学習自体をAIとの対話で進める形にしてはどうか」** という案が出たため、UI/UX を視覚的に検証するためのプロトタイプを作成しました。

## 検討ペルソナ

26歳・非IT業種営業職。未経験からIT職への転職志向。**夜間限定・短時間**で学習。明確なロードマップと客観評価で動機付けられる。

## 最重要KPI

**学習完遂率**

## 設計スタンス（重要）

完全対話化は採用しない。理由は対話の認知負荷が読むより高く、夜疲れたペルソナでむしろ脱落リスクが上がるため。代わりに **「対話メイン + 教材は同期スライドで残す」ハイブリッド** を採用。

## 主要な UI/UX 変更点

| 項目 | 現行版 | 提案版 |
|---|---|---|
| メイン動線 | 教科書を読む | KANAMEとの対話で進む |
| 教材表示 | 左メイン | 右ペインに同期表示 |
| 進度 | スクロール量 | 対話の流れ＋上部Layer積み重ね |
| 入力 | ほぼなし | チップ中心 + 自由入力（補助） |
| 詰まった時 | 自分でなんとかする | 「もう一度」「比喩で」「次へ」の3つの逃げ道 |
| レッスン終了 | 次へボタン | 3点要約（✓/△/×）+ Layer 1枚積む演出 |

## このプロトタイプで体験できること

1. **チップで7ターンの対話進行** — クリックで会話が進む
2. **同期スライド** — KANAMEの発言に応じて右ペインが自動切替
3. **理解度バー** — 対話進行で伸びる
4. **Layer 積み上げ演出** — レッスン終了時に塗り重ねが1枚増える
5. **現行版/提案版トグル** — 上部で Before/After 比較
6. **3つの逃げ道** — 各ターンで選択肢に「とりあえず次へ」「あとで戻る」が常時ある

## 採用しているコア原則

- **個別カスタマイズAI**: 営業比喩エンジン（サーバ＝倉庫担当 など）
- **1つの動線**: チップは常に3つ以下、メイン動線をブレさせない
- **One Thing at a Time**: 1ターン = 30秒以下のチャンク
- **漆メタファー**: Layer 積み上げ可視化＝塗り重ねの厚み = 進捗
