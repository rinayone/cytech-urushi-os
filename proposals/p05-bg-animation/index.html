P-05 背景アニメーション デモ

炎モチーフ（volcano-bg.png）に下から立ち昇る動きを試したデモ。3パターン横並びで比較できます。
背景配置は P-05 本線と揃えています（モチーフ左寄せ・右側暗）。

PATTERN A

静止（現状ベースライン）

サーバとは何か
読む 3分 / フェーズ 1 / 7

動きなし。集中阻害ゼロ、リソース消費もゼロ。比較用ベースライン。

animation: なし
PATTERN B（推奨）

控えめに揺れる炎

サーバとは何か
読む 3分 / フェーズ 1 / 7

下から微かに立ち昇る + 微小な色相揺らぎ。10秒周期 で ease-in-out。意識を引かないが、見ていると確かに「生きている」感じが伝わる。学習集中への影響は最小限。

animation: 10s / translateY ±1.5% / scaleY 1.015 / hue-rotate 6° / brightness 1.06
PATTERN C

派手に燃える炎

サーバとは何か
読む 3分 / フェーズ 1 / 7

大きく伸び縮み + 色相回転 + 透明度ゆらぎ。6秒周期、3キーフレーム。インパクトあるが、学習中の集中阻害リスクあり。スカウト解禁画面など「特別な瞬間」用に検討の余地あり。

animation: 6s / translateY ±3% / scaleY 1.05 / hue-rotate ±25° / opacity 0.45→0.6

判断材料

  • 背景アニメーションの効用: 漆メタファーの「層が動く＝生きている」感を演出可能。「学習が止まらない」というブランド体験の補強
  • リスク: 学習中の集中阻害。背景が動くと視線が引かれる可能性（特に夜間・疲労時の佐藤くんペルソナ）
  • 推奨: 採用するなら Pattern B（控えめ）。Pattern C は P-05本線では避けて、スカウト解禁などの特別画面用にとっておく
  • パフォーマンス: いずれもCSS変形（GPU加速対象）+ filter で実現。重い処理はないが、モバイルではバッテリー影響を確認推奨
  • 採用判断は本線適用前に社長レビューでの印象を確認: Mockの段階では「試した」状態に留めるのが安全
※ このデモは縮小ステージ（16:10）。実機では viewport いっぱいに広がるため、揺らぎ幅は相対的に小さく見えます。
