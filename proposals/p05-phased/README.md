## 提案の経緯

1. v1.1 P-05 = 教科書型（読むパートだけ）
2. p05-dialogue 提案 = 全部対話型
3. **本案 = フェーズ切替型** ← 「家庭教師は対話だけで教えていない」という気付きから

## 設計の核

家庭教師の本来のレッスンは「**読む → 対話 → 読む → 対話 → 演習 → まとめ**」という構造。1モードに固執せず、フェーズごとにUIを切り替えるのが自然。

## 1レッスン構成（合計 約8分）

| # | フェーズ | 時間 | UI モード |
|---|---|---|---|
| ① | イントロ | 30秒 | 対話（KANAME挨拶 + 今日の進め方） |
| ② | 読む | 3分 | 教科書モード（v1.1風） |
| ③ | 確認 | 1分 | 対話モード（2-3ターン） |
| ④ | 読む | 2分 | 教科書モード（コードなど） |
| ⑤ | 確認 | 1分 | 対話モード（2ターン） |
| ⑥ | 演習 | 1分 | クイズモード（4択 + 即フィードバック） |
| ⑦ | まとめ | 30秒 | 振り返りモード（3点要約 + 任意の自由入力 + Layer積み上げ） |

## 主要UI/UX要素

### フェーズインジケーター（上部）
7段の進度バー。各段に対応するアイコン（📖💬✏️など）と残り時間が見える。**「今どのフェーズか」が常時可視化** → ペルソナの「ロードマップ重視」性向と整合。

### モード別レイアウト
- **読むモード**: 教材中心、920pxキャップで読みやすい行長
- **対話モード**: チップを **会話インラインで右寄せ**、画面下部固定にしない（自然な返答位置）
- **演習モード**: 1問4択、即フィードバック、解説付き
- **振り返りモード**: 要約カード + 任意自由入力（ポートフォリオ連携）

### KANAMEは常時呼べる
フッターの「☰ / K」ボタンから右側スライドパネル（目次 / KANAMEチャット）が開ける。**家庭教師が横にいる感**を維持。

### 戻る・進むナビ
誤操作時にフェーズ間を行き来可能。各フェーズインジケーターを直接クリックでもジャンプ可能。

### 自由入力は1日1回だけ（任意）
振り返りフェーズでだけ「今日の気付きを1行」を入力。スキップOK。書くと P-07 ポートフォリオに残る動線（実装時）。

### チップは答えを色で示さない
推奨選択（primary）も他の選択肢と同じ視覚スタイル。色で正解を教えない設計。「考えて答える」プロセスを残し、定着率を担保。

### フッター進捗バー: 完走で7色レインボー
CTUガイドライン7色のレインボーが空間に固定され、進度に応じて右からめくれて現れる。100%到達 = 自然にフルレインボーが完成（漆メタファー: 塗り重ね完了の演出）。

### コンテンツ最大幅 920px
全シーン共通。バブルは 540/480px キャップで対話の親密感維持。大型モニタでも会話が伸びすぎない。

## 「全部選択 vs 自由入力」議論への回答

- 対話パート内のチップ操作は**全部選択でOK**（負荷を下げる）
- アウトプット定着は**演習パート**が担う（手を動かす）
- 自由入力は **振り返りで1日1回だけ**（強制しない）
- → 役割分担が明確になり、無理に対話に全部背負わせる必要がなくなる

## このプロトタイプで体験できること

1. **7フェーズを自分のペースで進める** — 戻る/進む/インジケータークリック
2. **モード切替の感覚** — 読む→対話→演習で画面構造が変わる
3. **対話パートのチップ進行** — 不正解時のリアクション含む
4. **演習パートのフィードバック** — 正解/不正解に応じた解説
5. **Layer積み上げ演出** — 振り返りフェーズで層が増える
6. **KANAMEを呼ぶ** — 読み中・演習中も右上から呼び出せる
