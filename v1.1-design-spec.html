1. エグゼクティブサマリー

v1.0は全ポータル（求職者/企業管理者/TNG管理者）のプロトタイプを構築した。v1.1では求職者ポータルに特化し、「学校」ではなく「家庭教師」としてのUX設計に刷新する。

v1.1 コアコンセプト

一人ひとりに合った提案をする「家庭教師」体験。ログイン後のファーストビューはAIとの1対1の対話画面。前回の行動に基づいたパーソナライズ提案から、学習・選考・アクションへ自然に誘導する。
最重要KPI：学習完遂率

4. カスタマージャーニー

フェーズ1：不安の払拭と目的の点火（Ignite & Compass）

LM原則： Ignite / Compass

体験：ログイン直後のチュートリアルで簡単な質問に答えると、「佐藤さんには『情シス』が向いています」と提示。企業検索画面で「採用ロードマップ」が表示され、「このカリキュラムを推定○○時間学習し100%完了すれば面接確約」という明確なゴールが可視化される。ネガティブ思考がポジティブな動機へ反転する。

対応ページ： P-01（適性診断）/ P-04（カリキュラム検索）/ P-08（選考管理）

フェーズ2：学習の実行と見取り（Challenge & Reflect）

LM原則： One Thing at a Time / Reflect

体験：1回の学習セッションは情報量が絞り込まれている。専門用語でつまずくとAIメンターが「営業の仕事に例えると〜」と助言。学習が数日止まるとAIから能動的に声がかかり、孤独感・挫折を防ぐ。

対応ページ： P-02（AI対話）/ P-03（ダッシュボード）/ P-05（学習実行）

フェーズ3：実践とAI評価（Apply & Assessment）

LM原則： Apply / 第2層AI判定（Rubric基準＋根拠開示）

体験：実際のトラブル事例を解決する実技課題を提出。KANAMEから「佐藤さんのこの回答はB1レベルを満たしています」と根拠付きフィードバックが返る。客観的評価が自信を生む。

対応ページ： P-06（課題・テスト提出）

フェーズ4：学習完了と実力証明（Skill Prime達成）

LM原則： B1到達 / Fair

体験：カリキュラム完遂。学習時間・AI評価ログが蓄積された「嘘のつけない分厚い履歴書」と「スキルグラフ」が完成。「B1レベルの実務適性がある」という確固たる自信を得る。

対応ページ： P-07（ポートフォリオ）

フェーズ5：マッチングと選考（Hiring & Revenge）

LM原則： Skill Prime（学んでから入る）

体験：採用試験ゲートのロックが解除され、希望企業へエントリー可能に。履歴書公開設定により、学習プロセスを評価した企業からログ引用スカウトが届く。「自分が選ぶ側にもなれる」体験。

対応ページ： P-08（選考管理）/ P-08（スカウト）

5. Learning Module v2.1（要約）

Beyond Study.

体験の質は我々の責任、経験の深さは本人の成果。学ぶべき知識は全員に等しい。しかし届け方は一人ひとり異なる。使えば使うほどフィットする「生きたカリキュラム」が設計の核心。

5.1 学びの原則

原則内容実装ガイド
Ignite（点火）「なぜ学ぶのか」を自分ごとにする。記憶定着率1.5〜2倍向上開始3〜5分以内に「こんな場面で困ったことは？」を入れる
Challenge（山場）正解率70〜80%の「ちょうど良い負荷」正解率に応じて難易度を動的調整
Reflect（見取り）自信と実力のズレに気づかせる「自信度→正解率比較」を標準装備
Recall（根付き）「思い出す」行為で記憶を強化復習タイミングを履歴に基づき動的決定
Apply（活かす）知識を「使える状態」へ移行各学習単位の最後に「実務への適用課題」

5.2 設計の原則

原則内容
One Thing at a Time1セッションで導入する新概念は原則1つ。全設計判断の最優先
Why学ぶ理由が自分のこととして見えている
Agency自分で選んで進んでいる実感がある
Alignment目標・学習・評価が1本の線でつながっている
Compass今どこにいるか、次に何をすべきかが常にわかる
Spiral同じテーマを深さと間隔を変えて繰り返す

5.3 レベリング（B1〜B5）

レベル呼称観察可能な行動
B1Rookie手順があれば作業を完了できる
B2Player定型業務を一人で遂行・報告できる
B3Pro未経験の課題を自ら調べ解決できる
B4Expert判断基準を設計し他者に展開できる
B5Master新手法を創出し組織外にも影響を与える

Skill Primeとの関係： Skill Prime完了 = B1到達。「職場で働く準備ができた」証明。

5.4 評価（3層構造）

層担い手役割
第1層AI学習中のフィードバック（育てる）。即時に大量に返す
第2層AI修了判定（認定する）。基準に基づく判定と根拠の開示
第3層人間異議申し立て時の個別審査

5.5 Skill Prime（採用モデル）

  • 二値判定： 完了か未完了の二値。中間状態なし
  • 積み上げ式： Module単位で独立。取得済みは失効しない
  • Fair実装： 受験資格はModuleの完了のみ。年齢・学歴・職歴は条件に含めない

6. v1.1 設計方針

6.1 「家庭教師」設計思想

観点v1.0（学校）v1.1（家庭教師）
ファーストビューダッシュボード（統計一覧）AI対話（パーソナライズされた声がけ）
学習への誘導ユーザーが自分でページを探すAIが次のアクションを提案し、ボタン1つで遷移
パーソナライズなし（全員同じ画面）前回行動に基づく提案（離脱検知・進捗・時期に応じた文言）

最重要KPI—学習完遂率

6.2 ナビゲーション設計

要素役割
メインハブP-02 AI対話（ログイン後のファーストビュー。全遷移の起点）
ダッシュボード左サイドバー（260px）＋右メインエリアの2カラム構成。サイドバーに「Your Layers」（レベル・バッジ・カレンダー・アカウントメニュー）、メインにKANAME吹き出し＋デイリーミッションカード＋サジェストチップ＋展開パネル。ポートフォリオ・企業選考・設定・通知はポップアップで表示
常時ヘッダーどの画面からでもAI対話・ダッシュボードに戻れる最低限の導線。「学習に戻る」ボタン
設定パネルサイドバー下部のアカウントメニューからアクセスするポップアップ。プロフィール（アイコン変更機能付き）・パスワード・通知・リマインダー・言語設定

6.3 全体ルール

ルール内容
初期対応はAIのみサービス利用開始から学習完了まで、人間のカウンセリングは行わない。全てAI（KANAME）が対応する。人が関わるのは企業からのスカウト時・面談時から。
スキルの積み上げ別カリキュラムで習得済みのスキル（例：HTML/CSS）は引き継がれ、不足分のみ学習すればよい。カリキュラムを変更しても、過去の学習は無駄にならない。
カリキュラム同時受講制限受講中にできるカリキュラムは1つのみ。検索・閲覧は自由。別カリキュラムに変更する場合は現在のものを中断する。
採用枠の競争可視化学習者は、採用枠に対して自分が全体の何%にいるかを確認できる。早い者勝ちの構造を動機付けに活用する。※ペルソナにとって信憑性が低い場合は表示方法を要検討。
面接不合格→再チャレンジ面接不合格の場合、再学習が可能（振り出しに戻る）。ただし学習履歴は全て保持される。
スカウトは隠し機能P-08スカウト画面は常時表示ではなく、スカウトの条件を満たした時のみ「特殊イベント」として解放される。学習に集中させるため。学習していないユーザーが面接に直接進む導線は作らない。
AI名称現在「KANAME」だが、リリース前に正式名称を決定予定。AIキャラクターの口調・言葉遣いもリリース前に統一する。
最重要KPI用語「学習率」ではなく「学習完遂率」が正しい用語。完了する・マスターするという意味。
ボタンカラールールオレンジ→ピンク(#f7513b→#f13884): 最重要CTA（学習再開・次へ進む・KANAMEと始める）
パープル→ブルー(#953ec3→#0f4bab): 確定・送信系（提出・保存・PDF出力）
ブルー→ライトブルー(#0f4bab→#34e0ff): ナビゲーション（ダッシュボードへ・戻る系）

8. ページ別 詳細設計

P-01 適性診断（初回フロー）

目的

「キャリアの迷子」状態の佐藤くんに、「自分に向いているIT職種」を提示し、ネガティブをポジティブに反転させる。

LM原則

Ignite（なぜ学ぶのか）/ Compass（現在地と次の一手）

主要要素

  • 簡単な質問フロー（興味・経験・働き方の希望）
  • KANAME AIによる適性職種の提示（例：「情シスが向いています」）
  • 提示された職種に対応するカリキュラム＋推定学習時間のプレビュー
  • 「このカリキュラムを完了すれば面接確約」のゴール提示
  • → P-02（AI対話）またはP-04（カリキュラム検索）への誘導

P-02 AI対話（ファーストビュー）

目的

「家庭教師」のハブ。家庭教師が「ピンポーン」と家に来て、会話の中から自然に復習・学習に入る体験。ユーザーは「勉強しなきゃ」と思う前に、気づいたら学習の中にいる。

LM原則

Ignite / Compass / Recall

ビジュアル設計

  • 背景： 黒（#050508）。背景装飾は今後検討
  • KANAME キャラクター： CyTechカラーのレインボーグラデーションオーブ（中央配置）。ダーク面＋白い目の顔
  • 点火演出： ページ読込で自動発火。オーブがスケール0.2→1.0にポップ＋炎フラッシュ。スタートボタンなし・音声なし

UI構成

要素役割デザイン
AIメッセージ（3つ）挨拶→肯定→問いかけの順で順次表示ガラスモーフィズムバブル（角丸・左上角は小さめ）
選択肢（3つ）1ラリーで学習画面へシームレス遷移アウトラインボーダー＋左に丸インジケーター。AIバブルと明確に区別
チャットアイコン補助導線。必要な人だけ使う右下に控えめなアイコン。タップで入力パネル展開
ヘッダー唯一の出口「ダッシュボードへ →」ボタンのみ

1ラリーフロー（学習中ユーザーの例）

メッセージ→選択肢→学習へシームレス遷移

KANAME①：「おかえり、佐藤さん。」
KANAME②：「前回はネットワーク基礎まで進みましたね。」
KANAME③：「営業先の組織図と同じ構造なんですが、この図だとどこが"本社"に当たると思います？」
選択肢： サーバーかな / ルーターだと思う / うーん、わからない
→ どれを選んでも「そうそう、いい感じです」→ Unit 5の中にいる

文言のトーン

  • 常にポジティブ。離脱期間やネガティブな数字には触れない
  • 「前回はここまでできましたね」の肯定から入る（軽いさわり）
  • 「学習を再開する」→「続きから始める」（勉強感を消す）
  • トーン：「先生に呼び出された」ではなく「友達がおかえりと言ってくれた」

動線設計

  • 出口は1本のみ。ユーザーを迷わせない
  • CTAは1つ（状態に応じて内容が変わる）
  • チャットアイコンは控えめに配置（メイン動線を邪魔しない）
  • CTA ＞＞ チャットの優先度を視覚的に明確化

P-03 ダッシュボード（ゲームUI＋漆テーマ）

目的

LM原則

Compass / Reflect / Agency

P-04 カリキュラム検索

目的

「ロードマップ不在」の不安を解消。推定学習時間とゴールを可視化し、「届く」実感を与える。

LM原則

Alignment / Why

制約ルール

カリキュラム同時受講制限

受講中にできるカリキュラムは1つのみ。学習中であっても他のカリキュラムの検索・閲覧は自由に行える。新しいカリキュラムを受講開始するには、現在のカリキュラムを完了するか中断する必要がある。

主要要素

  • 推定学習時間の表示（「あと○○時間で面接確約」）
  • Skill Primeゲートと紐づくカリキュラム＋合格基準の可視化
  • 企業別の採用ロードマップ表示
  • 受講中 / おすすめ / スキル別のフィルター
  • 受講中カリキュラムの明示表示（1つのみアクティブ）

P-05 学習実行

目的

未経験者でも挫折しない、情報量を絞った学習体験。家庭教師コンセプトを忠実に再現するため、1レッスン内で「読む→対話→演習→振り返り」のフェーズを切り替える構成。

LM原則

One Thing at a Time / Challenge / Ignite

主要要素

  • フェーズ切替型レッスン（約8分／7フェーズ）: イントロ → 読む3分 → 確認1分 → 読む2分 → 確認1分 → 演習1分 → まとめ
  • 上部フェーズインジケーター（現在地と残時間を常時可視化）
  • 各フェーズ最適UI: 読む=教科書モード / 確認=KANAME短対話（2-3ターン）/ 演習=ミニ問題1問 / まとめ=3点要約＋振り返り入力
  • KANAMEは常時呼び出し可能（スライドパネル）
  • 戻る・進むナビでフェーズ間を行き来可能（誤操作リカバリ）
  • 振り返り入力はポートフォリオに自動保存（任意）
  • 完了時に Layer +1 演出 → 「理解度チェックへ →」で P-06 へ遷移

P-07 ポートフォリオ（キャリアポートフォリオ）

目的

「証明の壁」を取り除く。学習ログ・AI評価が蓄積された「嘘のつけない履歴書」。企業応募時の履歴書としても機能する。

LM原則

Fair / Skill Prime

主要要素（人物情報 / 履歴書）

  • 基本情報: 氏名（漢字+ふりがな）、生年月日、年齢（自動算出）、性別、居住地、連絡先、志望分野
  • 自己PR: 自由記述（200〜400字推奨、空欄時はCTA表示）
  • 学歴: リスト型（学校名・学部・学科・入学/卒業年月）。複数追加・削除可
  • 職歴: リスト型（会社名・部署・役職・在籍期間・業務内容、在職中フラグ）。複数追加・削除可
  • 資格・語学: チップ型リスト（取得年月任意）。複数追加・削除可

主要要素（学習成果）

  • B1〜B5スキルグラフ
  • 獲得バッジ一覧
  • 学習時間・AI評価ログの蓄積表示
  • カリキュラム進捗・正答率・プロフィール閲覧数

編集UI

  • 「プロフィール編集」ボタン → タブ式モーダル（基本情報 / 自己PR / 学歴 / 職歴 / 資格）
  • 各セクションヘッダーに ✎ アイコン → モーダルの該当タブを直接開く（クイック編集）
  • 各リスト項目に ✎ / 🗑 アイコン → 個別編集・削除（hover で表示）
  • 削除時は確認ダイアログ（必須項目の氏名・連絡先等は編集のみ可、削除不可）

PDF出力（3形式から選択）

  • 📄 履歴書: JIS規格相当の標準フォーマット（基本情報＋学歴＋職歴＋資格）
  • 📝 職務経歴書: 業務経験を詳述（職歴＋自己PR＋資格＋CTU学習成果）
  • 🎨 ポートフォリオ: CTUカラフル形式（学習成果＋スキルグラフ＋評価ログ推し）

公開設定

  • ポートフォリオ全体の公開/非公開トグル（企業のスカウト・閲覧許可）
  • 項目別公開制御は管理側で運用（ユーザー操作には含めない）

機能要件メモ（REG画面実装時に対応）

  • 新規会員登録（REG）で入力済みの基本情報（氏名・生年月日・メール等）は、ポートフォリオに自動反映する
  • REGで聞かない項目（学歴・職歴・自己PR・資格）はポートフォリオで後追い入力
  • 登録ハードルを下げるため、REGでは最小項目（名前・メール・パスワード・生年月日）のみ収集

P-08 スカウト

目的

「選ぶ側にもなれる」体験。企業が学習ログを引用してスカウトする機能。学習が進むほどスカウトが増える構造により、学習完遂率への動機付けにもなる。

LM原則

Skill Prime / Fair

主要要素

  • 受信スカウト一覧（企業名・ポジション・引用ログ）
  • 企業が引用した学習ログ・課題修正プロセスの表示
  • スカウトへの返信・辞退機能
  • ポートフォリオ公開設定との連携
  • プロフィール閲覧数表示（スカウト前段階の指標。「あなたのプロフィールを閲覧した企業」）
  • 評価ポイントの可視化（どの学習ログが企業に刺さったかを表示）
  • スカウト受信の推移グラフ（学習が進むほどスカウトが増えることを可視化）
