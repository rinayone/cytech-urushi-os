1. エグゼクティブサマリー

v1.0は全ポータル（求職者/企業管理者/TNG管理者）のプロトタイプを構築した。v1.1では求職者ポータルに特化し、「学校」ではなく「家庭教師」としてのUX設計に刷新する。

v1.1 コアコンセプト

一人ひとりに合った提案をする「家庭教師」体験。ログイン後のファーストビューはAIとの1対1の対話画面。前回の行動に基づいたパーソナライズ提案から、学習・選考・アクションへ自然に誘導する。
最重要KPI：学習完遂率

2.2 v1.0の課題

  • 全ポータルを網羅した「広く浅い」プロトタイプ
  • 求職者体験が「学校」的な一方通行のUI（ダッシュボード→自分で探す構造）
  • AIメンターはチャットFABのみ。パーソナライズ・能動的声がけがない
  • 学習完遂を促す設計（離脱防止・動機維持）が不足

課題心理v1.1で解決する方法

4. カスタマージャーニー

フェーズ1：不安の払拭と目的の点火（Ignite & Compass）

LM原則： Ignite / Compass

体験：ログイン直後のチュートリアルで簡単な質問に答えると、「佐藤さんには『情シス』が向いています」と提示。企業検索画面で「採用ロードマップ」が表示され、「このカリキュラムを推定○○時間学習し100%完了すれば面接確約」という明確なゴールが可視化される。ネガティブ思考がポジティブな動機へ反転する。

対応ページ： P-01（適性診断）/ P-04（カリキュラム検索）/ P-08（選考管理）

フェーズ2：学習の実行と見取り（Challenge & Reflect）

LM原則： One Thing at a Time / Reflect

体験：1回の学習セッションは情報量が絞り込まれている。専門用語でつまずくとAIメンターが「営業の仕事に例えると〜」と助言。学習が数日止まるとAIから能動的に声がかかり、孤独感・挫折を防ぐ。

対応ページ： P-02（AI対話）/ P-03（ダッシュボード）/ P-05（学習実行）

フェーズ3：実践とAI評価（Apply & Assessment）

LM原則： Apply / 第2層AI判定（Rubric基準＋根拠開示）

体験：実際のトラブル事例を解決する実技課題を提出。KANAMEから「佐藤さんのこの回答はB1レベルを満たしています」と根拠付きフィードバックが返る。客観的評価が自信を生む。

対応ページ： P-06（課題・テスト提出）

フェーズ4：学習完了と実力証明（Skill Prime達成）

LM原則： B1到達 / Fair

体験：カリキュラム完遂。学習時間・AI評価ログが蓄積された「嘘のつけない分厚い履歴書」と「スキルグラフ」が完成。「B1レベルの実務適性がある」という確固たる自信を得る。

対応ページ： P-07（ポートフォリオ）

フェーズ5：マッチングと選考（Hiring & Revenge）

LM原則： Skill Prime（学んでから入る）

体験：採用試験ゲートのロックが解除され、希望企業へエントリー可能に。履歴書公開設定により、学習プロセスを評価した企業からログ引用スカウトが届く。「自分が選ぶ側にもなれる」体験。

対応ページ： P-08（選考管理）/ P-09（スカウト）

5. Learning Module v2.1（要約）

Beyond Study.

体験の質は我々の責任、経験の深さは本人の成果。学ぶべき知識は全員に等しい。しかし届け方は一人ひとり異なる。使えば使うほどフィットする「生きたカリキュラム」が設計の核心。

5.1 学びの原則

原則内容実装ガイド
Ignite（点火）「なぜ学ぶのか」を自分ごとにする。記憶定着率1.5〜2倍向上開始3〜5分以内に「こんな場面で困ったことは？」を入れる
Challenge（山場）正解率70〜80%の「ちょうど良い負荷」正解率に応じて難易度を動的調整
Reflect（見取り）自信と実力のズレに気づかせる「自信度→正解率比較」を標準装備
Recall（根付き）「思い出す」行為で記憶を強化復習タイミングを履歴に基づき動的決定
Apply（活かす）知識を「使える状態」へ移行各学習単位の最後に「実務への適用課題」

5.2 設計の原則

原則内容
One Thing at a Time1セッションで導入する新概念は原則1つ。全設計判断の最優先
Why学ぶ理由が自分のこととして見えている
Agency自分で選んで進んでいる実感がある
Alignment目標・学習・評価が1本の線でつながっている
Compass今どこにいるか、次に何をすべきかが常にわかる
Spiral同じテーマを深さと間隔を変えて繰り返す

5.3 レベリング（B1〜B5）

レベル呼称観察可能な行動
B1Rookie手順があれば作業を完了できる
B2Player定型業務を一人で遂行・報告できる
B3Pro未経験の課題を自ら調べ解決できる
B4Expert判断基準を設計し他者に展開できる
B5Master新手法を創出し組織外にも影響を与える

Skill Primeとの関係： Skill Prime完了 = B1到達。「職場で働く準備ができた」証明。

5.4 評価（3層構造）

層担い手役割
第1層AI学習中のフィードバック（育てる）。即時に大量に返す
第2層AI修了判定（認定する）。基準に基づく判定と根拠の開示
第3層人間異議申し立て時の個別審査

5.5 Skill Prime（採用モデル）

  • 二値判定： 完了か未完了の二値。中間状態なし
  • 積み上げ式： Module単位で独立。取得済みは失効しない
  • Fair実装： 受験資格はModuleの完了のみ。年齢・学歴・職歴は条件に含めない

6. v1.1 設計方針

6.1 「家庭教師」設計思想

観点v1.0（学校）v1.1（家庭教師）
ファーストビューダッシュボード（統計一覧）AI対話（パーソナライズされた声がけ）
学習への誘導ユーザーが自分でページを探すAIが次のアクションを提案し、ボタン1つで遷移
パーソナライズなし（全員同じ画面）前回行動に基づく提案（離脱検知・進捗・時期に応じた文言）
離脱防止なしAIが能動的に声がけ（「3日ぶりですね」「前回の課題で詰まっていませんか？」）
最重要KPI—学習完遂率

6.2 ユーザーフロー

初回ユーザー: ログイン → P-01 適性診断 → P-02 AI対話 → P-03 ダッシュボード / P-05 学習
学習中ユーザー: ログイン → P-02 AI対話（パーソナライズ提案）→ 学習 / 課題 / アクションへ
学習完了: ログイン → P-02 AI対話（選考・スカウト提案）→ P-08 選考 / P-09 スカウトへ

8. ページ別 詳細設計

P-01 適性診断（初回フロー）

目的

「キャリアの迷子」状態の佐藤くんに、「自分に向いているIT職種」を提示し、ネガティブをポジティブに反転させる。

LM原則

Ignite（なぜ学ぶのか）/ Compass（現在地と次の一手）

主要要素

  • 簡単な質問フロー（興味・経験・働き方の希望）
  • KANAME AIによる適性職種の提示（例：「情シスが向いています」）
  • 提示された職種に対応するカリキュラム＋推定学習時間のプレビュー
  • 「このカリキュラムを完了すれば面接確約」のゴール提示
  • → P-02（AI対話）またはP-04（カリキュラム検索）への誘導

P-02 AI対話（ファーストビュー）

目的

「家庭教師」のハブ。全ユーザーのログイン後のファーストビューとして、パーソナライズされた提案から次のアクションへ導く。

LM原則

Compass / Agency（自分で選んで進む実感）

主要要素

  • AIメンターとの対話UI（チャット形式）
  • 前回行動に基づくパーソナライズメッセージ
    • 例：「前回Unit5で手が止まっていましたね。CSSセレクターの部分を営業の顧客分類に例えると…」
    • 例：「3日ぶりですね。前回の課題、あと少しで完了です」
    • 例：「Skill Prime達成おめでとうございます！エントリー可能な企業が3社あります」
  • アクションボタン群（状況に応じて動的に変化）
    • 「学習を再開する」→ P-05
    • 「課題を提出する」→ P-06
    • 「スキルグラフを確認」→ P-07
    • 「エントリー可能な企業を見る」→ P-08

P-03 ダッシュボード

目的

学習進捗・統計の俯瞰。AI対話の補完として「自分の現在地」を数値で確認できる画面。

LM原則

Compass / Reflect

主要要素

  • 学習進捗（%）/ Skill Primeゲートまでの残り
  • スキル別進捗バー
  • 次の面接・選考ステータス
  • AI伴走メッセージ（躓き検知→声がけ）
  • ネクストアクション一覧

P-04 カリキュラム検索

目的

「ロードマップ不在」の不安を解消。推定学習時間とゴールを可視化し、「届く」実感を与える。

LM原則

Alignment / Why

主要要素

  • 推定学習時間の表示（「あと○○時間で面接確約」）
  • Skill Primeゲートと紐づくカリキュラム＋合格基準の可視化
  • 企業別の採用ロードマップ表示
  • 受講中 / おすすめ / スキル別のフィルター

P-05 学習実行

目的

未経験者でも挫折しない、情報量を絞った学習体験。

LM原則

One Thing at a Time / Challenge / Ignite

主要要素

  • 1セッション1概念の教材表示
  • AIメンターチャット（24時間即レス、営業経験などに例えた説明）
  • 正解率に応じた難易度動的調整
  • Ignite: 冒頭に「こんな場面で困ったことは？」

P-06 課題・テスト提出

目的

学んだ知識を「使える」状態に移行させ、客観的な実力評価を得る。

LM原則

Apply / 第2層AI判定 / Reflect

主要要素

  • 実技課題（実際のトラブル事例ベース）
  • KANAME AIフィードバック（B1〜B5レベル判定＋根拠開示）
  • 自信度 vs 正解率の比較表示
  • 思考プロセスの記録

P-07 ポートフォリオ（キャリアポートフォリオ）

目的

「証明の壁」を取り除く。学習ログ・AI評価が蓄積された「嘘のつけない履歴書」。

LM原則

Fair / Skill Prime

主要要素

  • B1〜B5スキルグラフ
  • 獲得バッジ一覧
  • 学習時間・AI評価ログの蓄積表示
  • ポートフォリオPDF出力
  • 公開/非公開設定（企業へのスカウト許可）

P-08 選考管理

目的

Skill Prime達成後、ゲート解除→エントリーまでの導線。採用ロードマップの可視化。

LM原則

Skill Prime / Compass

主要要素

  • 採用ロードマップ（エントリー→書類→面接→内定のステップ可視化）
  • Skill Primeゲート解除ステータス
  • 必須課題100%完了者のみエントリーボタン活性化
  • 選考中企業のステータス一覧

P-09 スカウト

目的

「選ぶ側にもなれる」体験。企業が学習ログを引用してスカウトする機能。

LM原則

Skill Prime / Fair

主要要素

  • 受信スカウト一覧（企業名・ポジション・引用ログ）
  • 企業が引用した学習ログ・課題修正プロセスの表示
  • スカウトへの返信・辞退機能
  • ポートフォリオ公開設定との連携
