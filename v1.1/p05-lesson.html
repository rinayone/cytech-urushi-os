ネットワーク構成図の読み方

◉営業先の取引先一覧を思い出してみてください。本社→支店→部署と階層がありますよね？ネットワークもまったく同じ構造です。

ネットワークは、情報が流れる道です。営業で言えば「本社の指示が支店に届き、支店から各部署に伝わる」のと同じように、サーバーからの情報がルーターを経由して各端末に届きます。

＝ 本社 ＝ 支店 ＝ 部署

サーバーは「本社」にあたります。ルーターは「支店」のように情報を振り分けます。実際にどのようにデータが流れるか、コードで確認してみましょう。

network_check.py 
# ネットワーク構成を確認するスクリプト
import socket

# サーバー（本社）のIPアドレスを取得
hostname = socket.gethostname()
ip_address = socket.gethostbyname(hostname)

print(f"サーバー名: {hostname}")
print(f"IPアドレス: {ip_address}")

# ルーター（支店）へのルート確認
gateway = "192.168.1.1"
port = 80

try:
    sock = socket.create_connection((gateway, port), timeout=3)
    print(f"ルーター {gateway} に接続成功 ✓")
except:
    print("接続できません")

このように、サーバーのIPアドレスを取得し、ルーターへの接続確認を行います。営業先リストで「この支店に電話が繋がるか確認する」のと同じ発想です。
