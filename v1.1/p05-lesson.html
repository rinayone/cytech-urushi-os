読む 3分 サーバとルーターの関係
●営業先の取引先一覧を思い出してみてください。本社→支店→部署と階層がありますよね？ネットワークもまったく同じ構造です。

ネットワークは、情報が流れる道です。営業で言えば「本社の指示が支店に届き、支店から各部署に伝わる」のと同じように、サーバからの情報がルーターを経由して各端末に届きます。

本社支店部署

サーバは「本社」にあたります。ルーターは「支店」のように情報を振り分けます。端末は各部署のPCのことです。

営業活動で例えるなら、本社（サーバ）に取引先データがあって、各支店（ルーター）が必要なデータを抜き出し、それを各部署（端末）に届ける流れです。

では、実際にネットワークのサーバとルーターがちゃんと繋がっているかを確認するコードを見てみましょう。営業先リストで「この支店に電話が繋がるか確認する」のと同じ発想です。

network_check.py 
# サーバ（本社）のIPアドレスを取得
import socket
hostname = socket.gethostname()
ip_address = socket.gethostbyname(hostname)

print(f"サーバ名: {hostname}")

# ルーター（支店）に接続できるか試す
gateway = "192.168.1.1"
try:
    sock = socket.create_connection((gateway, 80), timeout=3)
    print("ルーター接続OK ✓")
except:
    print("接続できません")

このコードは2つのことをやっています。
① サーバの名前とIPアドレスを取得 → 「本社の住所を確認」
② ルーターに接続できるか試す → 「支店に電話してみる」
